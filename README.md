# comp_methods_lab_1

Проект для лабораторной работы по вычислительным методам с разделением на 2 части:
- C++ backend: численное решение ОДУ методом Рунге–Кутта 4-го порядка;
- UI (Python + Tkinter + Matplotlib): запуск расчета, просмотр таблиц, построение графиков и фазового портрета.

Проект решает:
- тестовую задачу (с точным решением для сравнения ошибки);
- основную нелинейную задачу колебаний груза с демпфером и нелинейной пружиной;
- оба режима шага: фиксированный и адаптивный (с контролем локальной погрешности).

## Реализованные этапы

1. Каркас проекта (CMake + C++ + UI)
2. Решение тестовой задачи и основной задачи
3. RK4 в двух режимах:
	- фиксированный шаг
	- адаптивный шаг с контролем локальной погрешности
4. Вывод таблиц и сводных метрик
5. Графики:
	- тестовая: точное и приближенное решение
	- основная: `u(x)`, `u'(x)`
	- фазовый портрет `(u, u')`
6. Эксперименты влияния `k*` и `c`
7. Шаблон отчета

## Постановка задач

### Тестовая задача

\[
\frac{du}{dx} = (-1)^{n}\frac{n}{2}u, \quad n=25, \quad u(0)=u_0
\]

Для `n=25`: `λ = -12.5`, точное решение:

\[
u(x)=u_0 e^{-12.5x}
\]

### Основная задача

\[
m u'' + c u' + k u + k^*u^3 = 0
\]

Параметры:
- `k = 2.0 Н/см`
- `k* = 2.0 Н/см^3`
- `c = 0.15 Н·с/см^2`
- `m = 1.0 кг = 0.01 Н·с^2/см`
- `u(0)=10 см`
- `u'(0)=0`

Система первого порядка:

\[
y_1=u, \quad y_2=u', \quad
y_1'=y_2, \quad
y_2'=-\frac{c}{m}y_2-\frac{k}{m}y_1-\frac{k^*}{m}y_1^3
\]

## Метод

- Одношаговый метод Рунге–Кутта 4-го порядка.
- Для оценки локальной погрешности на шаге используется сравнение:
  - решение за шаг `h`;
  - решение двумя полушагами `h/2`.
- Оценка ОЛП:

\[
	ext{ОЛП} \approx \frac{\|v_{h/2} - v_h\|}{2^p-1}, \quad p=4
\]

- Для адаптивного режима:
  - если `ОЛП > eps`, шаг делится пополам (`C1++`);
	- если `ОЛП < eps/32`, шаг удваивается (`C2++`) при сохранении правой границы.

Также реализованы:
- контроль `Nmax`;
- контроль выхода на правую границу `b`.

## Формат таблиц

### Таблица 1 (тестовая)
`i, xi, vi, v2i, vi-v2i, ОЛП, hi, C1, C2, ui, |ui-vi|`

### Таблица 2 (основная)
`i, xi, vi, v2i, vi-v2i, ОЛП, hi, C1, C2`

## Сводные данные (в `result.json` и UI)

- `n`
- `b - xn`
- `max |ОЛП|`
- общее число удвоений и делений шага
- `max hi` и `min hi` с точкой `x`
- для тестовой: `max |ui - vi|` с точкой `x`

## Исследование влияния `k*` и `c`

Серии задаются во входном JSON:
- `kStarValues`
- `cValues`

UI-режим «Эксперименты (k*, c)» строит семейства кривых `u(x)`.
